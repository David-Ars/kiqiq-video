# Minimum Viable Annotation: The Coach's Workflow for High-Signal Match Review

Most amateur and academy coaches over-tag. A typical Sunday review timeline shows 180 events on a 90-minute match, half of which never get referenced again. Capture cost is high; information gain is low.

Minimum viable annotation (MVA) inverts the workflow. The coach defines the smallest set of tags that can answer the week's tactical question, tags only those, and discards everything else. The output is a 12-clip package the players actually watch, not a 200-event database nobody opens.

## What MVA Is Not

MVA is not a "lite" version of professional event tagging. Opta-grade event collection answers a different question (longitudinal benchmarking across thousands of matches). MVA answers a single question this week: *Did we restart pressure within 6 seconds of losing the ball in our middle third?*

The distinction matters because the workflow, the tools, and the success metric are all different. Confusing the two is how clubs end up paying for tracking data they cannot operationalise.

## The Three-Question Filter

Before opening any video software, the coach writes down three questions for the match. Each question must:

1. Reference a measurable behaviour (not a feeling).
2. Tie to next week's training plan.
3. Be answerable by tagging fewer than 25 clips.

A worked example for a back-three side struggling with wide overloads:

| Question | Behaviour | Tag |
|---|---|---|
| Did the wing-back step early enough on the opposite-flank switch? | Wing-back position at moment of switch | "WB-Switch" |
| Did the far-side CB rotate to cover? | CB lateral movement on switch | "CB-Rotate" |
| How many switches did we concede in zone 14 entries? | Switch followed by zone 14 entry | "Switch-Z14" |

Total tag types: 3. Total expected clips: 18 to 22. That is the entire annotation universe for the week.

## The Tool Layer

Coaches at sub-elite levels rarely need full broadcast-grade software. They need three things: frame-accurate clipping, a tag panel they can customise, and a way to export a single shareable package.

Pitch-side tools like [Metrica Sports PlayBase](https://www.metrica-sports.com?_go=david36) sit in this layer. PlayBase handles the MVA workflow without forcing a coach into a professional event-collection schema: custom tag buttons, telestration, and export to a shared cloud folder. For clubs already running a 3D tracking stack, the Nexus tier covers automated tracking integration. The point is not the brand; it is matching the tool tier to the question tier. Paying for Nexus when MVA is the workflow is the same category error as paying nothing and tagging in a notes app.

## What to Discard

MVA assumes most match footage has zero analytical value for *this week's* question. The coach should be willing to discard:

- Set-piece footage when the week's question is open-play pressing.
- Goal celebrations and crowd shots.
- Tactical sequences from the opposition unrelated to the three questions.

Discarding is harder than tagging because it feels like leaving information on the table. The trade-off is that a 12-clip review with high signal beats a 60-clip review the squad never watches.

## The Squad-Facing Output

The final deliverable is not the tag database. It is a sequenced clip reel, ordered by escalation:

1. Two clips of the behaviour done well (anchor).
2. Four to six clips of the behaviour failing (problem).
3. Two clips of the corrective pattern from training (resolution).

Twelve clips. Eight minutes of squad time. One tactical question answered.

## Next Step for the Coach

Pick one tactical question for the next match. Build the tag panel for that question only. If the export takes more than 90 minutes of post-match work, the tag set is too wide. Cut it in half.
